UCI 2123: Systems Engineering with AWS
Capstone Project
Introduction
The Capstone Project represents the culmination of your learning journey in the Systems Engineering with AWS course. Over the past 15 weeks, you have developed expertise across the full stack of modern web development, from foundational HTML, CSS, and JavaScript to backend Java services with Spring Boot, relational databases with SQL, and cloud deployment on AWS.

This project challenges you to synthesize all of these skills into a cohesive, production-grade application that demonstrates your readiness for future development roles. You will work independently to design, implement, test, deploy, and present a complete solution that addresses a real-world business problem.

The Capstone spans 7 class days and requires you to demonstrate technical proficiency, problem-solving ability, professional communication, and project management skills.

________________________________________
Project Objective
Design, develop, and deploy a full-stack web application that solves a meaningful business problem, demonstrating mastery of web development principles and AWS cloud deployment.
Learning Outcomes Assessed
Upon completion of this Capstone, you will have demonstrated the ability to:

1.	Design end-to-end web architectures integrating React front ends, Spring Boot APIs, and relational databases
2.	Implement robust data models, validation, and RESTful services using Java, SQL, and JPA/Hibernate
3.	Secure applications and infrastructure with authentication, authorization, and secrets management
4.	Automate build, test, and deployment workflows with CI/CD
5.	Deploy observable, reliable, and cost-aware cloud applications on AWS
6.	Apply AI tools responsibly to enhance development productivity
7.	Communicate technical decisions and business impact to stakeholders

Instructions
Phase 1: Planning & Design (Day 1)
1.1 Problem Definition
Select ONE of the following project options or propose your own (subject to instructor approval):

A.	Inventory Management System Build a system for a small business to track products, suppliers, and stock levels. Features include product CRUD operations, low-stock alerts, supplier management, and sales reporting.
B.	Employee Management Portal Create an HR portal for managing employee records, departments, leave requests, and performance reviews. Features include employee profiles, organizational hierarchy, and leave approval workflows.
C.	Event Booking Platform Develop a platform for creating and booking events. Features include event creation, attendee registration, ticket management, and event analytics.
D.	Custom Project (Instructor Approval Required) Propose your own project idea that demonstrates similar complexity and covers all required technologies.

●	Functional requirements (minimum 10)
●	Non-functional requirements (performance, security, scalability)
●	Out-of-scope items

●	Component Diagram for React frontend structure

Phase 2: Backend Development (Days 2-3)
2.1 Database Implementation
●	Create MySQL database schema using SQL scripts
●	Implement proper constraints (PRIMARY KEY, FOREIGN KEY, NOT NULL, UNIQUE)
●	Add indexes for frequently queried columns
●	Create seed data script for testing
2.2 Entity Layer
●	Create JPA entity classes for all tables
●	Configure relationships (@OneToMany, @ManyToOne, @ManyToMany)
●	Implement proper cascading and fetch strategies
●	Add validation annotations (@NotNull, @Size, @Email, etc.)
2.3 Repository Layer
●	Create Spring Data JPA repository interfaces
●	Implement custom query methods using @Query annotation
●	Add pagination support for list endpoints
2.4 Service Layer
●	Implement business logic in service classes
●	Use constructor injection for dependencies
●	Handle exceptions with custom exception classes
●	Implement transaction management with @Transactional
2.5 Controller Layer
●	Create REST controllers with appropriate mappings
●	Implement proper HTTP status codes
●	Add request validation with @Valid
●	Document endpoints with annotations
2.6 Error Handling
●	Create global exception handler with @ControllerAdvice
●	Return consistent error response format
●	Log errors appropriately

●	Add form validation with user-friendly error messages

●	Configure Spring Security filters

●	Hash passwords using BCrypt
●	Store secrets in SSM Parameter Store (not in code)
●	Implement CORS configuration
●	Add rate limiting for authentication endpoints
●	Validate and sanitize all user inputs
4.3 Unit Testing

●	Test both success and failure scenarios
●	Achieve minimum 70% code coverage
4.4 Integration Testing

●	Test authentication and authorization
●	Create Postman collection with all endpoints
4.5 Code Quality
●	Run SonarQube analysis

●	Ensure no security vulnerabilities
